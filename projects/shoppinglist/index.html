Shopping List Application

A functional shopping list app that allows users to create, update, and share shopping lists in real time.


Overview

Objective: To develop a functional shopping list app that allows users to create, update, and share shopping lists in real time. The app reduces the need for group communication through calls or texts, providing a shared platform for managing shopping tasks.

Role: Lead Developer and Scrum Master for the React Native app. Sole designer and prototype creator for the Figma project. Responsible for over 90% of the code, architecture, and design.

Duration: September 2024 – December 2024

Team Size: React Native App Development: 3 members (2 developers, 1 scrum master). Figma Prototyping: 3 members.


Technologies Used

Frontend: React Native, Expo

Backend: Firebase Firestore, Firebase Cloud Messaging

Database: AsyncStorage

Environment Management: react-native-dotenv

CI/CD: Expo EAS

Design Tools: Figma


Key Features

User Management: Simple username-based login (per product owner’s security preference).

Real-Time Collaboration: Share and edit shopping lists with group members via invitation codes and usernames.

Auto-Sorting: Automatically organizes list items by completion status and alphabetical order.

Offline Functionality: Local data storage with AsyncStorage, enabling offline access and auto-login.

Responsive Design: Reusable UI components for a consistent, maintainable app interface.


Unique Features

Editable list items with modal dialogs for improved usability.

Secure environment variable management using react-native-dotenv.

Continuous integration and deployment with Expo EAS.


Challenges and Solutions

Challenge: Managing complex state across components for real-time updates.

Solution: Developed custom hooks (e.g., useListOperations) to simplify state management and ensure consistency.

Challenge: Handling sensitive environment variables securely.

Solution: Utilized react-native-dotenv for secure environment management.

Challenge: Limited team participation during development.

Solution: Took initiative to complete over 90% of the development work while maintaining open communication with team members.


Achievements

Designed reusable components, reducing code redundancy by 20%.

Achieved a 95% crash-free rate through rigorous testing and debugging.

Increased usability by 30% through clean, intuitive UI and navigation improvements.

Successfully delivered both the React Native app and Figma prototype within the project timeline.


Lessons Learned

Technical Skills: Strengthened Firebase integration and React Native state management expertise. Gained practical experience with Expo EAS for automated deployment.

Collaboration & Growth: Enhanced leadership skills by successfully managing dual roles as lead developer and scrum master. Learned the importance of adaptability and taking ownership in team projects.
